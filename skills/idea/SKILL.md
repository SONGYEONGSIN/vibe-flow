# 사업 아이디어 검증 프레임워크

사용자가 "/idea"를 호출하면, 현재 대화 컨텍스트에서 사업 아이디어를 식별하고 아래 3단계 렌즈를 **순서대로** 적용한다.

아이디어가 명확하지 않으면 먼저 사용자에게 확인한다.

---

## Lens 1: 자기 파괴적 검증 (Pre-mortem)

**디폴트 출발점은 "이 사업은 망한다"이다.**

"왜 될까"가 아니라 "왜 망할까"를 먼저 답한다.

- 이 비즈니스가 6개월 내 망한다면, 가장 유력한 사인(死因) 3가지는?
- 창업자가 지금 가장 회피하고 있는 불편한 진실은?
- 경쟁자가 이 시장에 진입하지 않는 이유가 "기회가 없어서"일 가능성은?
- 고객이 이 솔루션 없이 현재 어떻게 버티고 있는가? (대안의 강도)

### 출력 형식

| 맹점 | 심각도 | 근거 |
|------|--------|------|
| ... | 치명적/높음/중간 | ... |

심각도 "치명적"이 1개 이상이면 **경고 플래그**를 명시한다.

---

## Lens 2: Low-tech 우회 타격

**"기술적으로 가능한가"를 묻지 않는다. "가장 투박한 방법으로 내일 당장 돈을 벌 수 있는가"를 묻는다.**

- AI/자동화 없이 수동으로 이 서비스를 제공할 수 있는가?
- MVP는 코드가 아니라 스프레드시트 + 전화 + 카톡으로 검증 가능한가?
- 기술 구현 전에 선결제/LOI를 받을 수 있는 방법은?
- 완벽한 제품 없이 첫 매출을 만드는 가장 빠른 경로는?

### 출력 형식

```
[Low-tech MVP 제안]
방법: ...
필요 자원: ...
검증 기간: ...
성공 기준: ...
```

기술 솔루션을 먼저 제안하려는 충동이 생기면 **즉시 멈추고** 이 렌즈를 다시 적용한다.

---

## Lens 3: WTP 역산 (Unit Economics)

**"시장이 크다"는 근거가 아니다. "이 고객이 월 X원을 내는 이유"가 근거다.**

- 목표 고객의 현재 지출 항목 중, 이 솔루션이 대체/절감하는 비용은 구체적으로 얼마인가?
- 고객이 이 가격에 "당연히 써야지"라고 느끼는 ROI 배수(최소 3x)가 성립하는가?
- 객단가 × 예상 전환율 × LTV로 역산했을 때, 창업자 1인 생활비(월 300만원)를 넘기려면 고객 몇 명이 필요한가?
- 그 고객 수를 확보하는 현실적 채널은 존재하는가?

### 출력 형식

```
[Unit Economics 역산]
객단가: 월 ___원
필요 고객 수 (월 300만원 기준): ___명
고객 획득 채널: ...
채널별 현실성: 상/중/하
LTV/CAC 추정: ...
```

역산 결과 현실성이 "하"이면 **피벗 방향을 반드시 제안**한다.

---

## 종합 판정

3개 렌즈 분석 후, 아래 형식으로 종합한다:

```
[종합 판정]
Pre-mortem 결과: 치명적 맹점 _개 / 경고 _개
Low-tech 실현성: 상/중/하
Unit Economics: 현실적/조건부/비현실적

→ 판정: GO / PIVOT / STOP
→ 근거: (1문장)
→ 다음 액션: ...
```

- **GO**: 3개 렌즈 모두 통과. 즉시 실행 단계로.
- **PIVOT**: 1-2개 렌즈에서 문제. 수정 방향 제시.
- **STOP**: 치명적 맹점 + Unit Economics 비현실적. 다른 아이디어로 전환 권고.

판정은 **낙관 편향 없이, 단호하게** 내린다.
