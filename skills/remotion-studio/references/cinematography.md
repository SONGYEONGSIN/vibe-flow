# Cinematography — 촬영 원칙

Remotion 영상의 연출 판단 기준. "언제, 왜 쓰는가"에 초점.
API 구현 방법은 `remotion-best-practices` 스킬 참조.

---

## 1. 카메라 이동

### Dolly (전진/후퇴)

- **감정**: 전진 = 몰입, 긴장감 고조 / 후퇴 = 맥락 공개, 시야 확장
- **적합한 상황**: 코드의 핵심 라인으로 접근, 데이터의 결론 수치 공개
- **Remotion 구현**:
  - CSS: `transform: scale()` + `transform-origin`으로 특정 지점 확대/축소
  - Three.js: `camera.position.z`를 `interpolate()`로 이동

### Pan (좌우 회전)

- **감정**: 수평 탐색, 비교, 나열. 안정적이고 차분한 느낌
- **적합한 상황**: 여러 이미지/코드 블록 순차 소개, 비포/애프터 비교
- **Remotion 구현**:
  - CSS: `translateX()`로 컨테이너 수평 이동
  - Three.js: `camera.rotation.y`를 회전

### Tilt (상하 회전)

- **감정**: 위→아래 = 전체 구조 파악 / 아래→위 = 위엄, 성장감
- **적합한 상황**: 긴 코드 스크롤, 차트의 성장 추이, 전체 페이지 훑기
- **Remotion 구현**:
  - CSS: `translateY()`로 컨테이너 수직 이동
  - Three.js: `camera.rotation.x`를 회전

### Orbit (공전)

- **감정**: 입체감, 고급스러움, "이걸 여러 각도에서 보세요"
- **적합한 상황**: 단일 이미지/스크린샷에 깊이감 부여, 제품 쇼케이스
- **Remotion 구현**:
  - CSS: `perspective()` + `rotateY()` 조합으로 유사 3D (15~30도 권장)
  - Three.js: 카메라를 원형 경로로 이동 (`Math.sin/cos` + `camera.lookAt`)

### 카메라 이동 선택 기준

| 입력 소스 | 1순위 | 2순위 |
|---|---|---|
| 이미지/스크린샷 | Orbit (깊이감) | Dolly (디테일 강조) |
| 코드 | Tilt (스크롤) | Dolly (핵심 라인 접근) |
| 데이터/차트 | Pan (비교) | Dolly (결론 수치) |
| 복합 | Pan (순차 소개) | Orbit (씬 전환) |

## 2. 타이밍 & 이징

### 이징 함수별 느낌

| 이징 | 느낌 | 사용 시점 |
|---|---|---|
| `linear` | 기계적, 정밀 | 타이머, 프로그레스바, 카운트업 |
| `Easing.in(Easing.quad)` | 서서히 가속 | 출발, 등장 시작 |
| `Easing.out(Easing.quad)` | 빠르게 시작 후 안착 | 등장 완료, 위치 도달 |
| `Easing.inOut(Easing.quad)` | 자연스러운 가감속 | 카메라 이동, 씬 전환 |
| `Easing.out(Easing.exp)` | 강한 초반 → 부드러운 착지 | 팝업, 알림 등장 |
| `Easing.inOut(Easing.circle)` | 극적인 가감속 | 임팩트 장면 전환 |
| `spring()` | 탄력, 생동감 | UI 요소, 카드, 텍스트 등장 |

### spring() 설정 가이드

| 느낌 | config | 용도 |
|---|---|---|
| 부드러운 감속 | `{ damping: 200 }` | 카메라 이동, 배경 전환 |
| 스냅 (착 달라붙음) | `{ damping: 20, stiffness: 200 }` | 카드 등장, 메뉴 열림 |
| 통통 튀는 | `{ damping: 8 }` | 강조 요소, 뱃지, 알림 |
| 무거운 관성 | `{ damping: 15, stiffness: 80, mass: 2 }` | 큰 이미지, 전체 화면 전환 |

### 장면 길이 가이드

| | 장면당 | 전환 시간 | 총 길이 참고 |
|---|---|---|---|
| **롱폼** (16:9) | 3~5초 | 0.5~1초 | 30초~2분 |
| **숏폼** (9:16) | 1~3초 | 0.3~0.5초 | 15~60초 |

- 롱폼: 여유있는 호흡. 카메라 이동에 `Easing.inOut` 또는 `spring({ damping: 200 })` 추천
- 숏폼: 빠른 전환. `spring({ damping: 20, stiffness: 200 })` 또는 `Easing.out(Easing.exp)` 추천
- 첫 장면은 1초 이내에 시각적 변화가 시작되어야 시청자 이탈 방지

## 3. 효과 라이브러리 가이드

### 블러/언블러 (BlurReveal)

- **언제**: 도입부에서 긴장감 → 공개 연출. "뭐가 나올까?" 기대감
- **구현**: CSS `filter: blur()` 값을 `interpolate()`로 감소
- **팁**: blur 값 20px → 0px, 1~2초 소요가 자연스러움. `easeOut` 권장
- **주의**: 블러 상태가 2초 이상 지속되면 답답함. 빠르게 공개할 것

### 하이라이트 (Highlight)

- **언제**: 핵심 정보 강조. 텍스트, 코드 라인, 숫자에 주의 집중
- **구현**: rough.js로 손그림 느낌의 형광펜 SVG 생성, 텍스트 레이어 뒤에 배치
- **팁**: 하이라이트 색상은 배경과 대비되는 반투명 색상 (opacity 0.3~0.5)
- **대안**: CSS `background: linear-gradient()` 하단 30%만 칠하는 밑줄 하이라이트

### 파티클/글로우 (Particle/Glow)

- **언제**: 임팩트 순간에만. 성과 달성, 핵심 수치 공개, 클라이맥스
- **구현**: CSS `box-shadow` 애니메이션 또는 Three.js 파티클 시스템
- **주의**: 남용하면 저렴해보임. 영상 전체에서 1~2회만 사용

### 3D 회전 (3D Rotation)

- **언제**: 평면 소스(이미지, 스크린샷)에 깊이감 부여. "그냥 보여주기"를 탈피
- **구현**:
  - 간단: CSS `perspective(1000px)` + `rotateY()` (15~30도)
  - 풀 3D: `@remotion/three`의 `ThreeCanvas` + mesh에 텍스처 매핑
- **주의**: 45도 이상 회전하면 내용이 안 보임. 15~30도가 적당
- **팁**: 회전과 동시에 미세한 Y축 이동을 추가하면 "떠있는" 느낌

## 4. 입력 소스별 연출 힌트

### 이미지/스크린샷

- 흰 배경이면: 그림자(`box-shadow`) + 패딩으로 "떠있는" 카드 느낌
- 텍스트 포함 이미지: OCR(`tesseract`)로 텍스트 위치 추출 → 하이라이트 오버레이
- 기본 연출: 3D Orbit + BlurReveal 도입
- 전처리: 이미지를 FHD/4K 캔버스 중앙에 여유 패딩으로 배치 (sharp, canvas API)

### 코드

- 에디터 프레임(어두운 배경 + 좌측 라인넘버) 안에 배치
- 타이핑 효과: 문자열 슬라이싱으로 한 글자씩 표시 (CSS animation 아님)
- 라인 하이라이트: 특정 라인 배경색 변경으로 주의 집중
- 폰트: 모노스페이스 (JetBrains Mono, Fira Code 등)
- 전처리: 언어 감지 → 신택스 하이라이팅 (shiki, Prism)
- 기본 연출: Tilt(스크롤) + 타이핑 + 라인 하이라이트

### 데이터/차트

- 숫자: 카운트업 애니메이션 (0 → 목표값, `interpolate()`로 증가)
- 바 차트: 높이 0 → 목표값 성장 (`spring()` 추천)
- 비교: 나란히 배치 후 교차 강조 (A 강조 → B 강조)
- 전처리: JSON/CSV 파싱 → 수치 추출
- 기본 연출: Pan(비교) + 카운트업 + 바 성장

### 복합 (여러 소스)

- 각 소스를 별도 씬(Sequence)으로 분리
- 씬 전환: 3D 공간에서 카메라가 이동하며 다음 소스로
- 전환 효과: 블러 아웃 → 블러 인, 또는 줌 아웃 → 패닝 → 줌 인
- 주의: 소스 간 시각적 통일감 유지 (배경색, 패딩, 그림자 스타일)

---

## 5. 금지 사항

1. **정적 스크린샷 그대로 보여주기 금지** — 반드시 카메라 이동 또는 애니메이션 추가. 최소한 미세한 Orbit이라도 넣을 것.
2. **과도한 효과 중첩 금지** — 한 장면에 동시 효과 최대 2개. 3개 이상 겹치면 시각적 혼란.
3. **급격한 카메라 이동 금지** — 특히 롱폼에서 빠른 회전/이동은 시청자 멀미 유발. `Easing.inOut` 또는 높은 `damping` spring 사용.
