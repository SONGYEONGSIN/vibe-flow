# Remotion Studio

입력 소스를 분석하여 역동적인 Remotion 영상을 생성하고, 성공한 결과를 템플릿으로 축적하는 영상 제작 스킬.

**핵심 철학**: 자동화가 아니라 축적. Claude의 창의성이 1차, 사용자의 판단이 2차, 축적된 템플릿이 3차 안전망.

**필수 톤**: "와, 멋있는데?" — 정적 스크린샷 보여주기는 금지. 반드시 카메라 이동과 구체적 애니메이션 포함.

## 참조 스킬

- **remotion-best-practices**: Remotion API 사용법 (interpolate, spring, Sequence 등). 코드 작성 시 반드시 참조.
- **references/cinematography.md**: 촬영 원칙 (카메라 이동, 타이밍, 효과). 스토리보드 설계 시 반드시 참조.

---

## 모드 판별

사용자 요청을 보고 3가지 중 하나를 선택:

| 요청 패턴 | 모드 |
|---|---|
| "이 이미지로 영상 만들어줘" | **모드 1: 새로 만들기** |
| "지난번 뉴스 기사 스타일로 만들어줘" | **모드 2: 템플릿 적용** |
| "이거 템플릿으로 저장해줘" | **모드 3: 템플릿 저장** |

혼합 요청("이 이미지로 지난번 스타일로 만들어줘")은 **모드 2**로 처리 — 템플릿 기반 + 새 입력.

---

## 모드 1: 새로 만들기

### Step 1 — 입력 분석

- 파일 확장자, 내용, 해상도, 배경색 등 파악
- 이미지에 텍스트가 포함되었는지 판단 (하이라이트/주석이 필요할 수 있음)
- 포맷(롱폼/숏폼): 입력 소스의 비율 등에서 추론 가능하면 추론 후 스토리보드에 포함, 불확실할 때만 질문

### Step 2 — 스토리보드 제안

references/cinematography.md를 읽고, 기존 templates/ 디렉토리에 유사한 템플릿이 있으면 참고하되 그대로 복사하지 않는다.

아래 형식에 맞춰 제시:

```
## 스토리보드 제안

**입력**: [파일 경로] ([설명, 해상도, 배경색 등])
**포맷**: [16:9 | 9:16 | 1:1] | [시간] | [fps]

### 장면 구성

| # | 구간 | 연출 | 효과 |
|---|---|---|---|
| 1 | 0~1초 | 전체 블러에서 시작, 서서히 선명해짐 (easeOut) | BlurReveal |
| 2 | 0~5초 | 이미지가 3D 공간에서 Y축 15도 천천히 회전 (easeInOut) | 3D Camera |

### 필요한 도구
- [도구명]: [용도]

### 분위기
[연출 의도와 근거 — 왜 이런 연출을 선택했는지]

> 이대로 진행할까요? 수정할 부분이 있으면 말씀해주세요.
```

**규칙:**
- 장면 테이블에 이징 정보 필수 포함 (linear, easeInOut, spring 등)
- 번호 부여로 "3번을 ~로 바꿔" 같은 구체적 수정 가능하게
- 사용자 승인 대기. 수정 요청 시 반영 후 재제시.

### Step 3 — Remotion 프로젝트 생성

**프로젝트 경로 관리:**
1. `config.md` 파일을 읽어 저장된 경로 확인
2. 없으면 사용자에게 프로젝트 위치를 물어보고, config.md에 저장

config.md 형식:
```markdown
---
project_path: /Users/username/remotion-projects
---
```

**프로젝트 생성/업데이트:**
- 해당 경로에 기존 Remotion 프로젝트가 있으면 새 Composition 추가
- 없으면 `npm create video@latest -- --blank`으로 scaffold
- 패키지 매니저는 기존 lockfile 감지 (yarn.lock → yarn, pnpm-lock.yaml → pnpm, 기본 → npm)
- components/ 디렉토리에 재사용 가능한 TSX가 있으면 프로젝트로 복사해서 활용
- remotion-best-practices 스킬을 참조하여 코드 작성

### Step 4 — 렌더링 & 확인

- `npx remotion render`로 MP4 출력 (Composition ID, 출력 경로 등 상세는 remotion-best-practices 스킬 참조)
- 사용자에게 결과 확인 요청

### Step 5 — 축적 제안

- "이 영상을 템플릿으로 저장할까요?" 물어보기
- 승인 시 → 모드 3 실행

---

## 모드 2: 템플릿 적용

### Step 1 — 템플릿 탐색

- templates/ 디렉토리의 frontmatter(tags, input_type)로 매칭
- 후보가 여러 개면 목록 보여주고 선택 요청
- 후보가 없으면 모드 1로 전환 안내

### Step 2 — 변형 확인

- 선택된 템플릿의 "변형 가능한 부분" 섹션을 사용자에게 보여주기
- 사용자가 바꿀 부분 확인 (최소한 source_path는 새 입력으로 교체)

### Step 3 — 코드 생성 & 렌더링

- 템플릿의 핵심 코드 + 도구 체인을 기반으로 생성
- 새 입력에 맞게 수정 (텍스트, 이미지 경로, 수치 등)
- 렌더링 (모드 1 Step 4와 동일)

---

## 모드 3: 템플릿 저장

진입점 2가지: 사용자가 직접 "템플릿으로 저장해줘"라고 요청하거나, 모드 1 Step 5에서 자동 제안. 두 경우 모두 동일한 절차를 따름.

### Step 1 — 추출

현재 영상의 스토리보드, 도구 체인, 핵심 코드, 의존성을 정리하여 아래 템플릿 형식에 맞춰 .md 파일 생성.

### Step 2 — 컴포넌트 분리 판단

- 기존 templates/를 스캔하여 동일한 효과가 이미 1개 이상 존재하는지 확인
- **2회 이상 반복되는 효과만** components/에 독립 TSX로 추출
- 첫 등장 효과는 템플릿 내 핵심 코드에만 포함, 분리하지 않음

### Step 3 — 저장

- `templates/{날짜}-{입력유형}-{효과키워드}.md`로 저장 (예: `2026-03-13-image-3d-rough-highlight.md`)
- 해당되면 components/에 추출한 컴포넌트도 저장

## 템플릿 형식

```markdown

name: [템플릿 이름]

tags: [효과 태그 목록]
tools: [사용된 외부 도구]
created: [날짜]
---

## 스토리보드

| # | 구간 | 시간 | 연출 |
|---|---|---|---|
| 1 | 인트로 | 0~1초 | ... |

## 도구 체인

1. [전처리 단계] — 예: `tesseract input.png stdout --psm 6` -> 바운딩박스 추출
2. [처리 단계] — 예: 이미지를 흰색 FHD 배경 위에 여유있게 패딩
3. [코드 생성] — 예: rough.js로 하이라이터 SVG 생성, 텍스트 레이어 뒤에 배치
