# 완독률 전략

## 이탈 데이터

| 지표 | 수치 |
|------|------|
| 2주 내 이탈률 | 70% |
| MOOC 완독률 | 7~59% |
| 온라인 학습 평균 완독률 | 6% (이탈률 94%) |
| 문단 5문장 이상 시 체류시간 감소 | 40% |
| 20,000단어 초과 시 | 이탈 급증 |

## 이탈 원인

1. **인지 부하:** 텍스트만 빽빽한 구성, 끝없는 스크롤
2. **시간 빈곤:** 직장/가사 등 다중 책임
3. **고립감:** 혼자 화면 보며 학습하는 외로움
4. **물리적 피로:** 스크린 장시간 노출

## 방어 전략

### 1. 모듈화 (가장 중요)
- 전체 8,000~15,000단어 권장
- 챕터별 1,000~2,000단어 (10~15분)
- 각 챕터마다 요약 + 시각 자료
- **모듈화만으로 완독률 70%까지 가능**

### 2. 모바일 가독성
- 문단 2~4문장 (50~150단어)
- 줄 간격: 폰트 크기의 1.5~2배
- 최소 1인치 이상 여백
- 시각적 휴식 공간 확보

### 3. 게이미피케이션
- 연속 기록(Streak): 킨들 사례 — 연간 독서량 2~3권 → 15~18권
- "하루 10페이지" 같은 작은 목표 + 뱃지 보상
- 진행률 표시 (프로그레스 바)
- 체크리스트로 달성감

### 4. 소셜 요소
- 소셜 주석 도구 사용 시 상호작용 5배 증가 (7회→36회/학기)
- 다른 독자의 진척도/댓글 실시간 노출
- 커뮤니티 연동 시 완독률 거의 100% 근접

### 5. 메타인지 제공
- 각 챕터의 예상 소요 시간 명시
- 전체 진행률 표시
- 난이도 사전 안내

### 6. 오디오+텍스트 결합
- 이머전 리딩 사용자 91%가 이해도/기억력 향상 보고
- 오디오만 대비 콘텐츠 소비량 월 2배 증가

## 전자책 적용 체크리스트

- [ ] 챕터가 2,000단어를 넘지 않는가?
- [ ] 각 챕터에 예상 소요 시간이 명시되어 있는가?
- [ ] 처음 2개 챕터에서 "빠른 성공 체험"이 있는가?
- [ ] 문단이 5문장을 넘지 않는가?
- [ ] 챕터 끝에 다음 챕터로 이어지는 장치가 있는가?
- [ ] 진행률을 시각적으로 확인할 수 있는가?
- [ ] 독자가 직접 체크할 수 있는 체크리스트가 있는가?
