# 구조 패턴 (Structure Patterns)

전자책 목차 아키텍처. 주제와 독자 유형에 맞는 구조를 선택하고, 챕터 수/순서/분량을 배분하는 가이드.

---

## 1. 4대 핵심 패턴

### 문제-해결 (Problem-Solution)

독자의 고통에서 시작해 해결책으로 끝나는 구조. 전자책에서 가장 범용적.

**6단계 트랜스포메이션 로드맵:**

| 단계 | 역할 | 비중 |
|------|------|------|
| 1. 긴급한 문제 정의 | 고통 건드리기, 공감대 형성 | ~10% |
| 2. 원인 분석 | 왜 이 상태에 머물렀는지, 부정(denial) 극복 | ~10% |
| 3. 마인드셋 준비 | 잘못된 가정 해체, 새 관점 제시 | ~15% |
| 4. 핵심 액션 플랜 | 단계별 해결책 (가장 많은 비중) | ~35% |
| 5. 트러블슈팅 | 슬럼프, 예외 상황, 주변 저항 극복 | ~15% |
| 6. 미래 확장 | 변화된 삶 시각화, 커뮤니티 연결 | ~15% |

**적합**: 자기계발, 실용서, 비즈니스, 독자 설득이 필요한 모든 경우.

### 프로세스 (Process / How-To)

목표 달성을 위한 순차적 단계. 이전 단계를 마쳐야 다음으로 넘어감.

**4MAT 학습 모델 적용:**

| 사분면 | 질문 | 챕터 역할 |
|--------|------|----------|
| 1 | Why? | 이 방법론이 왜 필요한지 동기 부여 |
| 2 | What? | 핵심 개념, 이론, 용어 정의 |
| 3 | How? | 단계별 실행 지침, 체크리스트 |
| 4 | What If? | 예외 상황, 트러블슈팅, 응용 |

**적합**: 기술 가이드, 튜토리얼, 워크북, 코스형 전자책.
**사례**: Atomic Habits — 4단계(신호-열망-반응-보상)를 목차 대구조로 사용.

### 원칙/주제별 (Principle-based / Thematic)

핵심 원칙들을 병렬 배치. 순서가 결정적이지 않아 독자가 선택적으로 탐색 가능.

**적합**: 에세이, 철학적 주제, 다각도 분석, 원칙 모음.
**주의**: 병렬이라도 난이도 순 배치는 필요. 쉬운 원칙 → 어려운 원칙.

### 스토리/경험 기반 (Narrative)

저자의 여정과 깨달음 중심. 실패담과 전환점이 핵심 무기.

**3단계 아키텍처:**

| 단계 | 분량 | 역할 |
|------|------|------|
| 도입 | 1~2장 | 독자 고통 구체화, 읽어야 할 당위성 |
| 본론 | 3~5장 | 방법론을 경험 기반으로 전개 + 장애물 극복 |
| 결말 | 1장 | 장기 로드맵과 비전 |

**적합**: 회고록, 창업기, 경험 기반 자기계발.

## 2. 세부 패턴

| 패턴 | 특징 | 적합 상황 |
|------|------|----------|
| 큐레이션 | 리소스 선별 + 맥락 연결 + 비판적 평가 | 도구 모음, 리소스 가이드, 무료 리드마그넷 |
| Q&A | 독자 질문 → 상세 답변 | FAQ, 복잡한 주제 쉽게 풀기 |
| 사례 연구 | 실제 성공/실패 사례 중심 분석 | 비즈니스, 마케팅 |
| 리스티클 | "N가지 팁/방법" 리스트 형태 | 가벼운 논픽션, 빠른 정보 습득 |
| 비교 | A vs B 대조 분석 | 기술 비교, 전략 비교 |
| 모듈식 | 각 장이 독립적, 순서 무관 | 레퍼런스, 사전, 발췌 읽기용 |

### 큐레이션형 분류 2가지

- **여정 단계별 (가로)**: 목적 달성 과정 순서로 리소스 배치
- **숙련도별 (세로)**: 입문 → 중급 → 고급 순 수직 배치

**핵심**: 단순 링크 나열이 아니라 맥락 연결 + 장단점 비판적 평가 + 맞춤 추천이 있어야 함.

## 3. 하이브리드 설계

전체 뼈대 1개 + 챕터 내부는 다른 패턴 결합 가능.

예시:
- 전체: **문제-해결** 6단계 → 챕터 내부: **프로세스**(How-To) 단계별 실행
- 전체: **스토리** 3단계 → 챕터 내부: **큐레이션**(경험 과정에서 쓴 도구 세트)
- 전체: **원칙별** 병렬 → 챕터 내부: **사례 연구**로 원칙 입증

## 4. 분량 가이드라인

### 무료 (리드마그넷)

| 항목 | 기준 |
|------|------|
| 전체 | 3,000~8,000단어 (10~30페이지) |
| 챕터 수 | 3~5개 |
| 챕터당 | 800~1,500단어 |
| 추천 패턴 | 문제-해결 (축약), 큐레이션, 리스티클 |

### 유료

| 항목 | 기준 |
|------|------|
| 전체 | 8,000~20,000단어 (50~150페이지) |
| 챕터 수 | 7~15개 |
| 챕터당 | 1,000~2,000단어 (노션/디지털), 최대 3,000단어 (PDF/단행본급) |
| 추천 패턴 | 문제-해결 6단계, 프로세스, 스토리 |

### 분량 배분 비율

```
도입부 (문제 정의 + 원인): ~25%
본론 (액션 플랜 + 실행): ~50%
결론 (트러블슈팅 + 미래): ~25%
```

### 일관성 규칙
- 챕터 간 분량 차이 2배 이내 유지 (7페이지 ↔ 30페이지 금지)
- 분량 맞추기보다 **"이 챕터가 목적을 달성했는가"**가 우선

## 5. 패턴 선택 의사결정

```
독자가 구체적 고통을 갖고 있는가?
  ├─ YES → 문제-해결
  └─ NO
      독자가 특정 기술/과정을 배우려 하는가?
        ├─ YES → 프로세스 (How-To)
        └─ NO
            저자의 경험/여정이 핵심 가치인가?
              ├─ YES → 스토리/경험
              └─ NO
                  여러 원칙/개념을 전달하는가?
                    ├─ YES → 원칙/주제별
                    └─ NO → 큐레이션 또는 리스티클
```

## 6. 목차 안티패턴

| 안티패턴 | 증상 | 체크 질문 |
|---------|------|----------|
| MECE 위반 | 분류 기준 혼용, 항목 중복/누락 | "한 계층이 같은 기준으로 나뉘었나?" |
| 과도한 세분화 | 목차가 3단계 이상 깊음 | "2~3단계 이내인가?" |
| 위계 역전 | 세부 사항이 대챕터급 | "다른 챕터의 하위여야 하지 않나?" |
| 장황한 제목 | 제목으로 내용 예측 불가 | "3초 안에 뭘 얻는지 아는가?" |
| 평면적 나열 | 모든 항목이 동일 중요도로 보임 | "시각적 위계가 있는가?" |
| 분량 불균형 | 챕터 간 3배 이상 차이 | "가장 긴 챕터 / 가장 짧은 챕터 < 2배?" |
