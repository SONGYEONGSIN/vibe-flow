# 변화 아크 (Transformation Arc)

전자책의 전략적 뼈대. 독자의 Before → After 여정을 설계하고, 이를 챕터 시퀀스로 변환하는 프레임워크.

---

## 1. 변화 설계 5단계

### 1단계: 독자 정의
이상적인 독자(Ideal Reader) 1명을 구체적으로 특정한다.
- 직업, 경험 수준, 일상에서 보고/듣고/느끼는 것
- "모든 사람을 위한 책" = 아무도 공감 못 하는 책 (보편적 독자의 오류)

### 2단계: Before State (현재 상태)
독자의 고통을 **3층 구조**로 분석한다.

| 층위 | 질문 |
|------|------|
| 외부적 | 독자가 직면한 가시적/물리적 문제는? |
| 내부적 | 그로 인한 감정적 고통(좌절, 불안, 두려움)은? |
| 철학적 | 이 문제가 존재하는 것 자체가 왜 부당한가? |

문제를 유발하는 근본 원인을 '빌런'으로 설정하여 초점을 맞춘다.

**저자 질문:**
- 독자의 가장 큰 과업(Job To Be Done)은?
- 현재 사용 중인 임시방편은 왜 실패하는가?
- 이 고통이 지속되면 최악의 결과는?

### 3단계: After State (변화 후 상태)
단순 문제 해결이 아니라 **열망적 정체성(Aspirational Identity)**을 정의한다.

정체성 변형 공식: `[현재 정체성] → [변화된 정체성]`
- "혼란스러운 초보자 → 유능한 전문가"
- "수동적 소비자 → 자기 결정권을 가진 창작자"

**저자 질문:**
- 독자가 궁극적으로 어떤 사람으로 변모하길 바라는가?
- 책 후 독자는 무엇을 소유(Have)하고, 어떤 감정(Feel)을 느끼고, 하루(Average Day)가 어떻게 달라지는가?
- 독자가 타인에게 어떻게 묘사되길 바라는가?

### 4단계: Bridge (브릿지) 설계
Before → After 사이의 간극을 메우는 해결책.

- 저자는 **가이드**로 포지셔닝 (영웅은 독자)
- 공감(독자의 고통을 안다) + 권위(해결할 능력이 있다) 입증
- 명확한 계획(Plan)과 행동 촉구(CTA) 제공

### 5단계: 목차로 변환 (Logic Mapping)
변화 로드맵을 챕터 시퀀스로 전환한다.
- 각 챕터 자체가 **미니 Before-After-Bridge** 구조
- 챕터를 읽어 나감에 따라 순차적으로 상태 변화가 일어나도록 배치

## 2. 백워드 디자인 (역순 설계)

> "가르쳤다고 해서 배운 것은 아니다" — Wiggins & McTighe

After State를 먼저 확정하고 역순으로 설계한다.

| 순서 | 단계 | 전자책 적용 |
|------|------|-----------|
| 1 | 바라는 결과 확인 | After State + 영속적 이해(Big Idea) 정의 |
| 2 | 수용 가능한 증거 결정 | 독자가 "나 할 수 있다"를 증명할 실습/체크리스트 설계 |
| 3 | 학습 경험 계획 | 실제 챕터 콘텐츠 작성 |

### 영속적 이해 (Enduring Understanding)
세부 사실을 잊어도 평생 기억할 핵심 아이디어. 각 챕터마다 1개씩 정의.

### 본질적 질문 (Essential Question)
단답형이 아닌, 호기심을 자극하고 탐구를 유도하는 질문. 챕터 도입부에 배치.

### 3단계 필터링
`친숙해질 가치가 있는 지식 → 중요한 지식 → 영속적 이해` 순으로 좁히기.
전자책에 모든 것을 넣지 않는다. 핵심만 남긴다.

## 3. 챕터 시퀀스 변환

### 5대 전환점

전체 전자책에서 독자가 도약하는 핵심 지점 5개를 먼저 배치한다.

| 전환점 | 위치 | 역할 |
|--------|------|------|
| 촉매제 | ~10% | 문제의 심각성 자각. "이대로면 안 된다" |
| 목표 설정 | ~25% | 변화 결심. 저자의 로드맵에 올라탐 |
| 미드포인트 | ~50% | Want(피상적 욕구) → Need(진짜 필요) 깨달음 |
| 어두운 밤 | ~75% | 과거 방식은 안 통하고 새 방식은 아직 미숙한 고군분투 |
| 클라이맥스 | ~90% | 변화 확정. 배운 것을 실제 적용하고 보상 획득 |

### 감정 장부 (Emotional Ledger)

모든 챕터에 대해 기록:

```
챕터 N: [제목]
  시작 감정: [지배적 기분/동기]
  결정적 사건: [이 챕터의 핵심 자극]
  종료 감정: [변화된 상태]
```

**시작 감정 = 종료 감정이면 죽은 챕터.** 삭제하거나 수정한다.

### Action/Reaction 교차 (Double Rainbow)

외적 사건(새 프레임워크, 새 도구) 챕터와 내적 성찰(적용, 회고) 챕터를 교대 배치하여 완급 조절.

### 페이싱 점검

챕터 제목만 나열했을 때 텐션 흐름이 보이는지 확인. 제목은 내용 요약이 아니라 아크의 진행 단계를 암시.

## 4. 안티패턴

| 안티패턴 | 증상 | 체크 질문 |
|---------|------|----------|
| 주제 ≠ 변화 | "리더십에 대한 책" (주제만 있고 변화 없음) | "구체적으로 뭐가 달라지는데?" |
| 보편적 독자 | "모두를 위한" | "이 책이 가장 급한 사람 1명은?" |
| 지식의 저주 | 전문 용어 남발, 기초 설명 생략 | "독자가 이 단어를 아는가?" |
| How 누락 | 주장만 있고 실행 단계 없음 | "읽고 나서 뭘 할 수 있는가?" |
| 자기중심 기획 | 저자 경험만 나열 | "So what? Who cares?" |
| 감정적 은폐 | 감동 일화로 논리 부재를 덮음 | "일화 빼면 논리가 서는가?" |
| 기발함 > 명확성 | 모호한 제목 | "제목만 보고 어떤 변화인지 아는가?" |
| Before/After 과대 | 비현실적 약속 | "증거 없이 100% 성공을 장담하는가?" |

## 5. 실행 워크시트

### A. BAB 워크시트 (Phase 0 직후 작성)

```
## Before
고통: [독자의 핵심 문제]
감정: [좌절감/불안/혼란]
파급: [문제가 일상에 미치는 영향]

## After
변화: [구체적으로 달라지는 것]
감정: [안도/자신감/통제감]
정체성: [From] → [To]

## Bridge
방법론: [저자의 해결 체계]
증거: [사례, 통계, 경험]
CTA: [독자의 첫 행동]
```

### B. 논픽션 북 맵 (Phase 1 목차 설계)

```
[변화 스토리보드]
현재 상태: ___
변화 후 상태: ___
도달 방법: ___

[챕터별 세부]
Ch.N:
  훅: [독자 주의를 끄는 장치]
  핵심 주장: [2~3개]
  스토리/사례: [주장을 입증하는 사례]
  상호작용: [성찰 질문, 워크시트, 체크리스트]
  전환: [다음 챕터로의 Open Loop]
  감정 장부: 시작[___] → 종료[___]
```

### C. 저자 점검 질문 (Phase 0 필수)

Before 파악:
1. 독자의 가장 큰 과업(JTBD)은?
2. 현재 임시방편은 왜 실패하는가?
3. 가장 큰 내부적 장애물(감정)은?
4. 해결 못 하면 최악의 결과는?

After 정의:
5. 읽은 후 독자는 어떻게 달라지는가?
6. 어떤 사람으로 변모하길 바라는가? (열망적 정체성)
7. 타인에게 어떻게 묘사되길 바라는가?
8. 무엇을 소유(Have)/느끼고(Feel)/하루가 어떻게 달라지는가(Day)?

Bridge 설계:
9. 3~4단계의 명확한 계획은?
10. 공감과 권위를 어떻게 증명하는가?
