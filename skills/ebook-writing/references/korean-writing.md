# 한국어 글쓰기 기준

## 문장

| 기준 | 수치 |
|------|------|
| 평균 길이 | 15~20단어 |
| 원칙 | 1문장 1아이디어 |
| 문체 | 능동형 우선 (피동형 "~되어지다" 지양) |
| 주어-서술어 | 최대한 가깝게 배치 |

## 문단

| 기준 | 수치 |
|------|------|
| 문장 수 | 2~4문장 (모바일 최적) |
| 모바일 한계선 | 3~5줄 초과 시 독자 이탈 시작 |
| 체류시간 감소 | 5문장 이상 → 40% 감소 |
| 강제 분할 | 10문장 이상이면 반드시 나누기 |
| 시각적 여백 | 문단 끝에 빈 줄 1개 |

## 전체 구조

| 기준 | 수치 |
|------|------|
| 황금 비율 | 서론 15% / 본론 70% / 결론 15% |
| 본론 분할 | 소주제별 3~4개 문단, 각 전체의 약 20% |
| 서론 한계 | 15% 초과 시 본론 진입 전 독자 지침 |

## 매체별 분량

| 매체 | 분량 |
|------|------|
| 블로그 포스팅 | 1,000~2,000자 (공백 제외) |
| 제목 | 20~25자 이내 (모바일 잘림 방지) |
| 전자책 챕터 | 1,000~2,000단어 (10~15분) |
| 전자책 전체 | 8,000~15,000단어 (완독률 최적) |
| 단행본 | 12만~15만자 (공백 포함) |

## 어휘

- 어려운 저빈도어 사용 조절 → 독해 난이도 낮추기
- 종결 어미 변화 → 문장 단조로움 방지 ("~합니다" 반복 금지)
- 전문 용어 사용 시 괄호로 쉬운 설명 병기
- 한자어보다 순한글 우선

## Slop 표현 금지 목록

| 금지 | 대안 |
|------|------|
| "자 이제~" | 바로 본론 시작 |
| "축하합니다!" | 삭제 또는 구체적 성과 언급 |
| "~에 대해 알아보겠습니다" | 핵심 질문으로 시작 |
| "어렵지 않습니다" | 삭제. 쉬우면 증명으로 보여줌 |
| "걱정하지 마세요" | 삭제. 구체적 해결책 제시 |
| "간단합니다" | 삭제. 간단하면 바로 보여줌 |
| 같은 내용 다시 요약 | 삭제. 한 번만 명확하게 |

## 종결 어미 변화 가이드

한 문단 내에서 같은 종결 어미가 3회 이상 반복되지 않도록:

```
❌ ~합니다. ~합니다. ~합니다.
✅ ~합니다. ~입니다. ~할 수 있습니다.

❌ ~해요. ~해요. ~해요.
✅ ~해요. ~거든요. ~할 수 있어요.
```
