# 노션 전자책 포맷 가이드

## 콜아웃 (Callout) — 시각적 앵커

**용도:** 정보 위계 설정, 독자 시선을 특정 지점으로 유도

| 유형 | 이모지 예시 | 용도 |
|------|-----------|------|
| 팁 | 💡 | 알면 유용한 정보 |
| 경고 | ⚠️ | 흔한 실수, 주의사항 |
| 핵심 | 🎯 | 챕터의 핵심 메시지 |
| 권바이브 노트 | 🗣️ | 저자의 사고방식/경험 |

**고급 활용:**
- 수학 공식 블록(`$$`)으로 콜아웃 첫 줄 텍스트를 크게 키워 챕터 도입부 시선 고정
- 인라인 페이지 → 다단(Multi-column) → 콜아웃 전환으로 내부 다단 레이아웃 구현
- 내부에 구분선, 이미지, 데이터베이스 삽입 가능

**규칙:**
- 한 챕터 내 콜아웃 3~4개 이하. 남용하면 시각적 앵커 효과 소실
- 성격별 이모지/색상 통일하여 독자가 패턴 학습하게 함

## 토글 (Toggle) — 인지부하 조절

**용도:** 방대한 정보를 숨기거나 노출하여 인지 부하 관리

**활용 패턴:**
- **퀴즈:** 질문만 노출, 정답/해설은 토글 내부 → Active Recall 유도
- **심화 자료:** 핵심만 노출, 코드 블록/로우 데이터/참고 문헌은 토글 내부
- **단계별 가이드:** 각 단계를 토글로 감싸서 순차적 공개
- **중첩 구조:** 토글 안에 토글/데이터베이스로 지식을 입체적 구성

**규칙:**
- 토글 제목만으로 내부 내용을 유추할 수 있어야 함
- 필수 내용을 토글에 숨기지 않기. 선택적 심화만 토글로
- 메인 흐름은 토글 밖에 유지

## 임베드 (Embed) — 인터랙티브 경험

**용도:** 정적 텍스트를 인터랙티브 작업 공간으로 전환

**활용 패턴:**
- **영상:** YouTube/Vimeo 삽입 → 페이지 이탈 없이 시각적 시연
- **위젯:** 진척도 바, 포모도로 타이머, 카운트다운
- **설문/퀴즈:** involve.me 등 연동하여 실시간 이해도 점검

**규칙:**
- 임베드는 보충 수단이지 본문 대체가 아님
- 영상 없이도 내용이 완결되어야 함
- 로딩 속도 고려: 한 페이지에 임베드 2~3개 이하

## 체크리스트 & 데이터베이스 — 실행 도구

**용도:** 수동적 소비 → 능동적 실행 전환

**활용 패턴:**
- **목표 트래커:** 레벨별 달성 체크리스트
- **회고 템플릿:** 각 챕터 끝에 "오늘 배운 것" 기록 공간
- **실습 체크리스트:** 미션의 단계별 완료 확인

**규칙:**
- 체크리스트는 5~7항목 이내. 너무 많으면 압도감
- 데이터베이스는 독자가 직접 채울 수 있는 구조로

## 페이지 구조 Best Practice

```
[콜아웃: 이 챕터를 끝내면 할 수 있는 것]

본문 (짧은 문단, 2~4문장)

[콜아웃: 팁/경고]

본문 계속

[토글: 심화 자료 / 코드 블록]

[임베드: 관련 영상 (선택)]

[콜아웃: 권바이브 노트]

[체크리스트: 미션]

[다음 챕터 예고 (Open Loop)]
```
