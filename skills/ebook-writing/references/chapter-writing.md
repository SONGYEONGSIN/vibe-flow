# 챕터 작성 가이드

챕터 단위 집필 시 참조. 유형별 템플릿, 깨달음 설계, 독자 유지 장치를 다룬다.

---

## 1. 챕터 템플릿 (유형별)

### 교육형 (Gagné 9 Events 기반)

튜토리얼, 코스형 전자책에 최적.

| 단계 | 역할 |
|------|------|
| Hook | SCQA 또는 PAS로 "왜 이게 필요한지" 도입 |
| 목표 제시 | 이 챕터를 끝내면 뭘 할 수 있는지 한 문장 |
| 선수 지식 연결 | 이전 챕터에서 배운 것과 연결 |
| 핵심 전달 | 개념 → 예시/시연 → 정리. Pyramid Principle(결론 먼저) |
| 실습 유도 | 따라할 수 있는 단계별 안내 |
| 피드백 | 잘 됐으면 이렇게 보여야 한다 / 안 됐으면 여기를 확인 |
| 정리 | 핵심 3줄 요약 |
| 다음 예고 | Open Loop — 다음 챕터 궁금증 유발 |

### 프레임워크형 (4MAT 기반)

방법론/원칙 전달에 최적. 각 챕터가 하나의 원칙을 전달.

| 사분면 | 질문 | 역할 |
|--------|------|------|
| Why | 왜 필요한가? | 개인적 의미와 동기 부여 |
| What | 그게 뭔가? | 핵심 개념, 이론, 용어 정의 |
| How | 어떻게 하는가? | 단계별 실행 지침, 체크리스트 |
| What If | 예외는? | 트러블슈팅, 엣지 케이스, 응용 |

### 스토리형

경험/사례 기반 전자책에 최적.

```
Scene → Tension → Insight → Application
장면 설정 → 갈등/위기 → 깨달음 → 독자 적용점
```

- 저자의 실패담과 전환점이 핵심 무기
- "나도 할 수 있다"는 효능감을 심어줌

### 논픽션 범용 (Fat Outline)

구조 패턴에 상관없이 대부분의 논픽션 챕터에 적용 가능.

```
Hook → 핵심 주장 2~3개 → 사례/스토리 → 실천/성찰 → 전환
```

## 2. 깨달음의 순간 설계

### 인지의 4단계

전자책 전체가 이 흐름을 따른다:

| 단계 | 독자 상태 | 챕터 역할 |
|------|----------|----------|
| 무의식적 무지 | "뭘 모르는지도 모름" | 기존 방식의 결함 노출 |
| 의식적 무지 | "내 방식이 틀렸구나" | 새로운 프레임 제시 |
| 의식적 숙련 | "이렇게 하면 되는구나" | 단계별 실습/적용 |
| 무의식적 숙련 | "자연스럽게 할 수 있다" | 체화/습관화 확인 |

### Dial vs Switch

| 유형 | 설명 | 가치 |
|------|------|------|
| Dial | 기존 틀 안에서 정보량만 증가 | 보충적 |
| Switch | 사고 틀 자체를 뒤집는 관점 전환 | 핵심 |

모든 전자책에 최소 1개 이상의 Switch 챕터가 있어야 한다. Switch가 없으면 "정보 모음집".

### 임계 개념 (Threshold Concept)

한 번 이해하면 되돌아갈 수 없는 변형적 아이디어. 각 챕터의 핵심 깨달음을 이걸로 설정.

### 깨달음 촉발 공식

1. **인지적 부조화 유발**: 기존 세계관과 모순되는 정보/사례 제시
2. **유추적 추론**: 전혀 다른 영역의 비유로 독자가 스스로 연결하게 함
3. **성찰 질문/워크시트**: 깨달음을 내재화하는 도구 제공 (WHERETO의 R)

## 3. 독자 유지 장치

### Slippery Slide (Sugarman)

모든 문장의 목적은 **다음 문장을 읽게 하는 것**.
- 첫 문장이 두 번째 문장을 읽게 하고, 두 번째가 세 번째를...
- 챕터 첫 문장에 가장 많은 공을 들인다.

### Open Loop

챕터 끝에 다음 챕터의 궁금증을 심는다.
- "이제 X를 할 수 있다. 하지만 Y 없이는 곧 벽에 부딪힌다. 다음 챕터에서..."
- Zeigarnik 효과: 미완결 과제가 기억에 더 오래 남음

### Curiosity Gap (Loewenstein)

알고 있는 것과 알고 싶은 것 사이의 간극. 이 간극이 독자를 끌어당김.
- 챕터 도입에서 질문을 던지고 → 본문에서 답을 준다
- 답을 주기 전에 충분한 긴장을 유지

### Action/Reaction 리듬

외적 사건(새 프레임워크, 새 도구) 챕터 → 내적 성찰(적용, 회고) 챕터 교대.
논픽션에서도 "새 개념 제시 → 독자 적용/성찰" 리듬을 유지.

## 4. Writing Formulas

| 공식 | 구조 | 용도 |
|------|------|------|
| SCQA | Situation → Complication → Question → Answer | 챕터 도입부 |
| PAS | Problem → Agitate → Solution | "왜 필요한가" 설명 |
| BAB | Before → After → Bridge | 챕터 인트로 |
| Pyramid | 결론 → 근거 → 세부 | 설명형 콘텐츠 |
| Promise-Deliver-Recap | 약속 → 이행 → 정리 | 챕터 전체 구조 |
| Open Loop | 궁금증 열기 → 나중에 닫기 | 챕터 간 연결 |

## 5. 챕터별 자가 점검

### 깨달음 점검 (매 챕터 필수)

- [ ] 독자가 믿고 있는 **잘못된 통념(Lie)**은?
- [ ] 챕터 끝에서 도달할 **새로운 진실(Truth)**은?
- [ ] 통념을 깨뜨릴 **결정적 사례/비유**가 있는가?
- [ ] What(무엇)을 알려주는가, How(어떻게)를 이끄는가?

### 감정 장부 점검

- [ ] 시작 감정 ≠ 종료 감정인가? (동일하면 죽은 챕터)

### 가독성 점검

- [ ] 문장 15~20단어, 1문장 1아이디어
- [ ] 문단 2~4문장 (5문장 이상 → 분할)
- [ ] 챕터 1,000~2,000단어 (유료 최대 3,000)

### 내용 점검

- [ ] One Idea Per Chapter
- [ ] Show Don't Tell — 주장 대신 사례/스크린샷
- [ ] So What? Test — "독자에게 뭐가 달라지는데?"
- [ ] Curse of Knowledge 방지 — 독자가 이 용어를 아는가?

### Slop 방지

- [ ] "자 이제~", "축하합니다!" 금지
- [ ] "~에 대해 알아보겠습니다" 금지
- [ ] 불필요한 요약 반복 금지
- [ ] 이미 설명한 것 다시 설명하지 않기
