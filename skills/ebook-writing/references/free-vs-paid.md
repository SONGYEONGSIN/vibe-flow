# 무료 vs 유료 캘리브레이션

무료(리드마그넷)와 유료 전자책의 콘텐츠 설계 차이. 같은 원칙이지만 깊이, 분량, 끝맺음이 다르다.

---

## 1. 핵심 비교

| 구분 | 무료 (리드마그넷) | 유료 |
|------|------------------|------|
| 목적 | 신뢰 구축 + 다음 상품 욕구 유발 | 완전한 변화 제공 |
| 변화 깊이 | Micro-Win (작은 성공 1개) | Paradigm Shift (패러다임 전환) |
| 초점 | What (문제 정의 + 해결 가능하다는 믿음) | How (전체 솔루션, 단계별 실행) |
| 아크 | Subtle (축약, 빠른 복귀) | Dramatic (혼돈 포함, 완전 완주) |
| 시간 | 15분 내 가치 체감 | 시간/인지적 투자 감당 |
| 끝맺음 | 의도적 간극 — 다음 문제를 드러냄 | 완전한 도구 소유 — 변화 완료 |
| 인지 4단계 | 1~2단계 (무지 → 자각) | 1~4단계 전체 (무지 → 숙련) |
| 분량 | 10~30페이지 (3,000~8,000단어) | 50~200페이지 (8,000~20,000단어) |
| 챕터 수 | 3~5개 | 7~15개 |

---

## 2. 무료 전자책 설계 원칙

### 구체성 편향 (Specificity Bias)

광범위한 변화가 아니라 **단일하고 시급한 문제 하나**에만 집중.
- "마케팅 전략 가이드" (X) → "인스타 릴스 조회수 1만 넘기는 3단계" (O)
- 범위가 넓을수록 리드마그넷으로서의 전환율이 떨어진다.

### 개념 증명 (Proof of Concept)

깊은 교육보다 **"이 방법이 통한다"는 증명**을 제공.
- 독자가 따라 해보고 "오, 진짜 되네?" 경험
- 15분 이내에 가치를 체감(Time-to-value)할 수 있어야 함

### 의도적 간극 설계

"관련성 높지만 불완전하게" 끝낸다.

**X without Y 패턴** (Hormozi):
- 문제 A를 완전히 해결 → 문제 B가 드러남 → 유료가 문제 B를 해결
- 첫 단계만 제공 → 전체 과정을 갈망하게 만듦
- 진단 도구 → 방치할수록 악화되는 문제를 독자가 스스로 깨닫게

**판매 타이밍**: 만족 시점이 아닌 **결핍 최고조 시점**.

### 추천 구조 패턴

| 패턴 | 예시 |
|------|------|
| 문제-해결 (축약) | 문제 정의 → 핵심 해결책 1개 → "더 있다" 암시 |
| 큐레이션 | "최고의 도구 10선" + 맥락/평가 |
| 리스티클 | "N가지 팁" 빠른 가치 전달 |
| 체크리스트/워크시트 | 독자가 직접 실행할 도구 |

---

## 3. 유료 전자책 설계 원칙

### 패러다임 전환

독자의 인식/행동 방식 자체를 근본적으로 바꾸는 **깊은 변형** 제공.
- 단순 지식 이해 → 완벽한 실행 능력(Competence) → 마스터(Mastery)
- 독자가 변화할 수 있다고 '믿는 것'을 넘어, 실제로 변화할 수 있는 **모든 도구를 소유**하게 만든다.

### 혼돈 단계 포함

리드마그넷처럼 '안전하고 빠른' 길이 아니라:
- 독자의 오래된 잘못된 믿음에 도전
- 변화의 비용에 직면하게 만드는 고군분투 구간 포함
- 완전한 변화 아크를 끝까지 완주

### How 전체 공개

'무엇'을 넘어 **'어떻게'**에 해당하는 모든 것을 숨김없이 제공:
- 전체 솔루션
- 단계별 튜토리얼
- 워크북/체크리스트
- 트러블슈팅
- 세부적인 논의와 엣지 케이스

### 추천 구조 패턴

| 패턴 | 예시 |
|------|------|
| 문제-해결 6단계 | 완전한 트랜스포메이션 로드맵 |
| 프로세스 (How-To) | 단계별 마스터 가이드 |
| 스토리 + 방법론 | 저자 여정 기반 실전 가이드 |

## 4. 무료 → 유료 연결 설계

```
무료 전자책
  │ 문제 A 해결 (Micro-Win)
  │ 문제 B 드러남 (의도적 간극)
  ▼
유료 전자책/상품
  │ 문제 B~Z 해결 (전체 변화)
  ▼
독자의 변화 완료
```

### 연결 체크리스트

- [ ] 무료가 유료의 "맛보기"가 아니라 **독립적 가치**를 갖는가?
- [ ] 무료 끝에서 독자가 **다음 문제를 자각**하는가?
- [ ] 무료에서 해결한 문제와 유료가 해결하는 문제가 **논리적으로 연결**되는가?
- [ ] 무료가 "이게 무료라고?" 수준의 **과잉 전달**인가?
- [ ] 유료가 무료의 단순 확장이 아니라 **깊이와 범위가 본질적으로 다른가?**

---

## 5. 안티패턴

| 안티패턴 | 문제 | 해결 |
|---------|------|------|
| 무료 = 대충 | 무료라서 품질 낮추면 브랜드 훼손 | "이게 무료라고?" 수준으로 |
| 무료가 완결적 | 유료로 연결될 이유가 없음 | 의도적 간극 설계 |
| 유료가 무료의 복붙 확장 | "돈 주고 살 이유가 없다" | 깊이/범위가 본질적으로 달라야 함 |
| 막다른 해결 | Core Offer로 연결 안 되는 리드마그넷 | 구조적으로 다음 상품과 연결 |
| 넓은 타겟 | "모두를 위한" = 아무도 반응 안 함 | 단일 문제, 단일 타겟 |
