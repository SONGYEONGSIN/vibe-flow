인수인계 작성폼 재구성(#858)에서 "운영가이드 레이아웃 참고" 요청을 선택/호버 색 표준만 이식(A안)으로 좁게 해석했다가, 사용자가 "달라진 게 없는데"라며 스크린샷 2장(원하는 모습 vs 현재)으로 재설명함 — 실제 기대는 박스 시트 제거 + 240px nav(2줄 항목) + 패널 헤더까지 구조 전체 이식(B안)이었다.

**Why:** 이 사용자에게 "레이아웃 참고"는 화면 골격(grid 구성, nav 형태, 헤더 톤)을 그대로 가져오라는 뜻. 색만 바꾸는 최소 변경은 "변화 없음"으로 인식된다.

**How to apply:** 레이아웃 참고 요청 시 (1) 기준 화면과 대상 화면 스크린샷/구조를 나란히 놓고 골격 차이를 먼저 제시, (2) 대안 제시 시 "색 표준만" 옵션을 추천으로 밀지 말고 구조 이식을 기본으로. 완료 검증은 claude-in-chrome으로 실제 렌더 확인이 효과적이었음. 관련: [[ops-console-dev-workflow]]
